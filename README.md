# voyageur_de_commerce
Dans le cadre du projet de TDLOG, mon GROUPE et moi allons travaillé sur un projet en Python visant à résoudre le problème du "Voyageur de commerce" (TSP - Traveling Salesman Problem). Ce problème d'optimisation consiste à déterminer le plus court chemin permettant à un voyageur de visiter un ensemble de villes, en passant une seule fois par chacune d'elles avant de revenir à la ville de départ.

Pour résoudre ce problème, nous allons mètre en place un algorithme utilisant des techniques d'optimisation basées sur des mutations et des inversions. L'algorithme itère un certain nombre de fois (), en ajustant les parcours possibles entre les villes pour améliorer progressivement le chemin. Des paramètres aléatoires ou prédéfinis influencent ces mutations afin d'explorer diverses combinaisons.

En fin de programme, des graphes sont générés pour visualiser le chemin initial ainsi que le chemin optimisé, permettant ainsi de constater l'amélioration réalisée. L'objectif de ce projet est non seulement de trouver la meilleure solution possible mais aussi de rendre le processus d'optimisation accessible et visuel grâce aux graphes.
